Allgemein

Als Renderer für die Markdown Eingaben kommt Marked zum Einsatz.

Dabei wird ein etwas veränderter Renderer sowie die Optionen GitHub Flavored Markdown gfm und Tabellen tables genutzt.
Ein gute Einleitung in die Syntax von Markdown befindet sich hier.
GitHub Flavored Markdown wird nicht vollständig untersützt.

	# Test
	Und hier dann der Text!!

	### Block

	> Und hier
	> `auch`!

Sonderfeatures

FontAwesome

Mittels :faclass: können Symbole aus FontAwesome angezeigt werden, :grin: zeigt z.B. einen fröhlichen Smiley.

FontAnwesome Symbole

Notizreferenzen

Mittels @noteid können können Notizen untereinder verlinkt werden. So wird mit @0bdd4aef0824abb4ff3f49dbd2ad7efc3e2545daa5afaf23b4cf6c0cb9a70d3b3e10a03ae84172d4f7f2e9dc46c662529783 z.B. ein Link zur Notiz 0bdd4aef0824abb4ff3f49dbd2ad7efc3e2545daa5afaf23b4cf6c0cb9a70d3b3e10a03ae84172d4f7f2e9dc46c662529783 erstellt.

Achtung, die NotizID wird nicht geprüft, sollte diese nicht vorhanden sein, so landet man in einer leeren und nicht speicherbaren Notiz.

Codeblöcke

	```lang
		// Code, Code, Code 
		if( $var ){
			do();
		}
	```
Wobei lang als Sprachangabe, siehe Liste, benötigt wird.
  • Syntaxhighlighting wird für Codeblöcke mit folgenden Sprachen untersützt:
    • Markup: markup
    • CSS: css
    • C: clike oder c
    • JavaScript: javascript
    • Bash: bash
    • C++: cpp
    • C#: csharp
    • Ruby: ruby
    • git: git
    • INI: ini
    • Java: java
    • JSON: json
    • Lua: lua
    • Markdown: markdown
    • Matlab: matlab
    • Objective C: objectivec
    • Perl: perl
    • PHP: php
    • Python: python
    • R: r
    • SQL: sql
    • Swift: swift
  • Weiterhin werden Codeblöcke mit folgenden Sprachen gerendert:
    • LaTeX (nur Mathe): tex
Alle weiteren Sprachen sowie Codeblöcke ohne Sprachangabe werden als Monospace in einem Rahmen dargestellt.
Basierend auf PrismJS und KaTeX
